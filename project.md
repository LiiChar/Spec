# Проект Spec

## Что это за проект

`Spec` — это desktop-приложение на `Rust` и `Dioxus 0.7.1`, которое отслеживает активность пользователя в Windows:

- определяет текущее активное окно
- фиксирует смену окна или состояние простоя
- сохраняет события в `SQLite`
- показывает историю активности в интерфейсе

Проект построен вокруг одного главного потока данных:

`Windows API -> tracker -> channels -> database + UI`

- `core` — доменная и системная логика

## Как это всё работает вместе

При запуске приложение проходит такой путь:

1. В [src/main.rs](E:/code/pet-project/Spec/src/main.rs:1) создаются три канала:
   - один для общего потока событий
   - один для записи в базу
   - один для обновления интерфейса
2. Отдельный поток запускает `tracker`, который регулярно опрашивает Windows и создает `EventModel`.
3. Промежуточный поток принимает каждое событие и дублирует его:
   - в поток записи в БД
   - в поток для UI
4. Поток базы данных сохраняет событие в `SQLite`.
5. Корневой UI-компонент при старте загружает старые события из базы, а потом начинает слушать поток новых событий.
6. Компоненты интерфейса получают список событий и отображают его как таймлайн или как список.

Итог:

- `core` производит данные
- `database` сохраняет данные
- `ui` читает и показывает данные
- `lib` помогает форматировать и агрегировать данные

## 1. Точка входа

Файл: [src/main.rs](E:/code/pet-project/Spec/src/main.rs:1)

Это главный orchestration-файл проекта. Он не содержит бизнес-логику трекинга или UI-логики, а связывает все части системы.

Что делает:

- инициализирует логгер Dioxus
- создает desktop-окно
- создает каналы для обмена событиями
- поднимает фоновые потоки
- запускает корневой UI-компонент `Root`

Именно здесь видно общую архитектуру приложения: сбор событий идет независимо от интерфейса, а UI только подписывается на уже готовый поток.

- `title`

- `process_path`

- `monitor_id`
- `timestamp`
- `duration`

## 3. Получение активного окна

Файл: [src/core/window/window.rs](E:/code/pet-project/Spec/src/core/window/window.rs:1)

Этот модуль работает напрямую с `Windows API`.

Что он делает:

- берет текущее foreground-окно
- читает заголовок окна
- получает размеры и позицию
- определяет PID процесса
- находит путь к процессу и имя процесса
- определяет класс окна
- определяет монитор
- собирает всё это в `WindowModel`

Это нижний системный слой проекта. Он ничего не знает про UI и почти ничего не знает про базу данных. Его задача только одна: получить актуальный снимок активного окна.

## 4. Трекинг активности

Файл: [src/core/tracker/tracker.rs](E:/code/pet-project/Spec/src/core/tracker/tracker.rs:1)

Это главный доменный модуль проекта.

Он отвечает за:

- регулярный опрос текущего окна
- определение состояния `idle`
- сравнение текущей активности с предыдущей
- завершение предыдущего интервала активности
- создание `EventModel`

### Как это работает

Трекер циклически:

1. проверяет, не бездействует ли пользователь
2. получает текущее окно через `get_current_window`
3. формирует `Activity`
4. сравнивает ее с предыдущей

Если активность не изменилась:

- трекер просто увеличивает длительность текущего интервала

Если изменилась:

- старая активность завершается
- по ней отправляется `EventModel`
- новая активность становится текущей

Таким образом проект хранит не “каждый тик опроса”, а интервалы активности.

## 5. База данных

Файл: [src/core/database/database.rs](E:/code/pet-project/Spec/src/core/database/database.rs:1)

Этот модуль отвечает за хранение истории.

### Что он делает

- создает таблицы при старте
- выполняет простую миграцию схемы
- сохраняет окна и события
- отдает события обратно в UI
- содержит несколько аналитических SQL-запросов

### Основные таблицы

#### `window_activity`

Содержит подробную информацию об окне:

- заголовок
- процесс
- размеры
- флаги состояния
- время

#### `events`

Содержит само событие:

- ссылка на запись окна
- тип события
- `timestamp`
- `duration`

### Зачем разделять окно и событие

Такое разделение позволяет:

- хранить событие как отдельный факт
- не смешивать структуру окна с типом активности
- строить выборки и аналитику по времени, типам и процессам

## 6. UI-слой

UI находится в папке [src/ui](E:/code/pet-project/Spec/src/ui:1).

Он делится на несколько уровней:

- `app` — корневой компонент приложения

- `widget` — крупные пользовательские блоки
- `components` — более мелкие переиспользуемые элементы

### Корень UI

Файл: [src/ui/app/root.rs](E:/code/pet-project/Spec/src/ui/app/root.rs:1)

`Root` — главный компонент интерфейса.

Что делает:

- подключает стили
- создает сигнал со списком событий

- слушает новые события из канала
- обновляет UI в реальном времени

То есть `Root` — это мост между фоновыми потоками и визуальными компонентами.

Файлы:

### Главный экран событий

Фактически это “страница аналитики активности”.
